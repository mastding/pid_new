# Project Instructions

## CodexSaver Delegation Policy

When the CodexSaver MCP tool is available, use it proactively for low-risk, bounded coding work in this repository.

Prefer CodexSaver for documentation, tests, code search, code explanations, lint/type-fix suggestions, boilerplate, and small mechanical refactors.

Do not delegate architecture decisions, security/auth/payment/permission work, database migrations, production operations, broad refactors, or final review. Use `dry_run` first if the task is ambiguous, and always review CodexSaver output before applying it.
