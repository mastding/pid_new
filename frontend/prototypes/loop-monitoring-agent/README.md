# PID Monitoring Agent Frontend Prototype

这是一个隔离的前端原型目录，用于设计“PID 智能整定驾驶舱”的工程化菜单和页面结构。

本目录不接入现有 React 路由，不修改现有前端功能，也不依赖后端接口。

## 文件

- `index.html`：可交互菜单化原型，参考“化工流程模拟智能助手”的顶部主菜单 + 左侧二级菜单风格。
- `script.js`：所有主菜单、子菜单、模拟整定数据、趋势图、诊断证据、弹窗内容的数据驱动渲染逻辑。
- `styles.css`：独立样式。
- `FRONTEND_SYSTEM_DESIGN.md`：详细前端系统设计，包括主菜单、子菜单、页面内容、字段、数据源、后端能力反推。

## 设计目标

目标不是“上传数据后跑整定”，而是“监控 Agent 持续理解回路状态，并在必要时触发诊断、辨识、整定建议”。

核心用户问题：

- 哪些回路当前不稳定？
- 当前是什么工况？
- 是否真的需要整定？
- 问题是否来自 PID 参数，而不是阀门、仪表、外扰或工况切换？
- 如果需要整定，数据是否足够支撑辨识？
- 如果不适合整定，下一步应该做什么？
- 参数建议能否下发，风险和回滚方案是什么？

## 当前原型已覆盖

- 7 个顶部主菜单可切换。
- 28 个子菜单页面可切换。
- 每个页面只显示当前子菜单内容，不再混合展示其他菜单说明。
- 代表性页面包含模拟趋势曲线、频谱图、知识图谱、KPI 卡片、数据表、门禁列表、诊断证据、整定参数候选。
- 无法在单页展开的操作通过弹窗模拟，例如“运行监控 Agent”“查看任务详情”“阻断原因”。
