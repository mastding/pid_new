2. 当前算法架构概览

2.1 Skill、Provider、Algorithm 的区别

层级 定位 典型目录 当前作用
Skill 面向 LLM / Agent 的原子能力封装，有输入模型、输出结果和上下文状态。 backend/core/skills/* 把加载数据、窗口检测、辨识、整定、评估等能力封成可调用动作。
Provider 可插拔算法实现单元，同一类能力可有多个 provider。 backend/core/providers/* 窗口算法族、模型拟合、整定策略、仿真评分、死区估计等确定性实现。
Algorithm 具体数学算法和历史入口，许多 provider 内部调用这里的函数。 backend/core/algorithms/* 系统辨识、PID 整定规则、闭环仿真、数据加载/评分等底层函数。
Pipeline 完整整定流程编排。 backend/core/pipeline/* 串联数据画像、本体/MCP、策略生成、窗口候选、LLM 评审、辨识、整定、评估。
Policy 工程约束和经验边界。 backend/core/policies/* 回路类型先验、最小合理时间常数、PID 参数物理边界、评分封顶规则。

2.2 当前主整定流程

历史数据导入/读取 → LoopFeatures 回路画像 → 本体/MCP 检索回路知识 → LLM/规则生成窗口策略 JSON → 策略驱动窗口算法族运行 → LLM 评审窗口候选与准入 → 多模型系统辨识 → LLM 模型评审与精修建议 → PID 参数整定 → 闭环仿真与性能评分

入口主要在 backend/core/pipeline/runner.py。窗口候选阶段可通过 stop_after="window_selection" 提前返回，用于前端“窗口候选”菜单。

4. LoopFeatures 回路画像指标与公式

核心实现：backend/core/shared/loop_features.py。该模块是当前“回路监控、单回路画像、窗口候选数据画像、整定准入”的主要基础特征来源。

4.1 数据基础画像

指标公式/规则说明
采样周期 时间戳相邻差值的中位数：median(diff(timestamp)) 同时输出 mean/min/max/std/P95/P99，用于判断不规则采样。
数据时长 duration_s = (N - 1) × sample_time_s 若有时间戳，也会输出开始/结束时间。
有效行数 PV/MV 有限值的行数。 SP/SV 可以不存在。

4.2 单变量统计：PV / MV / SP

均值: mean(x) 中位数: median(x) 标准差: std(x), 当前使用总体标准差 ddof=0 方差: var(x), 当前使用总体方差 ddof=0 跨度: span = max(x) - min(x) 变异系数: cv = std(x) / |mean(x)| 偏度/峰度: pandas Series.skew() / kurt() Pxx 分位数: percentile(x, xx) MAD: median(|x - median(x)|)

阶跃/跳变类统计基于相邻差分 dx = diff(x)，输出 P95/P99 绝对跳变、最大绝对跳变等。

4.3 数据质量 data_quality_raw

指标公式/阈值含义
缺失率 missing_ratio = NaN_count / N 分别统计 PV/MV/SP 和总缺失率。
重复时间戳 duplicated(timestamp).sum() 判断历史库导出是否存在重复点。
非单调时间戳 diff(timestamp) <= 0 时间倒序或重复都会影响时序算法。
不规则采样 |dt_i - sample_time| > max(1e-6, 0.2 × sample_time) 采样偏离中位周期超过 20% 记为不规则。
长间隔 dt_i > 3 × sample_time 历史数据断点或通讯中断。
常值比例 |diff(x)| <= 0.001 × span(x) 的比例 用于判断 PV/MV/SP 是否长时间不变化。
离群点数量 x < Q1 - 3IQR 或 x > Q3 + 3IQR 采用 3 倍 IQR，更偏保守。

4.4 MV 活动与操作强度

MV 活动阈值: move_threshold = max(0.03, percentile(|diff(MV)|, 60)) MV 活跃比例: active_step_ratio = mean(|diff(MV)| > move_threshold) MV 反向次数: direction_reversal_count = count(active[i] and active[i-1] and sign(dMV[i]) × sign(dMV[i-1]) < 0) MV 总行程: total_travel = Σ |diff(MV)|

4.5 PV/MV 关系与过程方向

实现函数：_pv_mv_relation_raw。为了降低单点噪声，PV/MV 先做 3 点滚动中位数，再计算水平量和差分量的相关性。

水平相关: corr_level(lag) = corr(MV[0:N-lag], PV[lag:N]) 差分相关: corr_diff(lag) = corr(diff(MV)[0:N-lag], diff(PV)[lag:N]) 搜索范围: lag ∈ [0, min(N/3, 3600s / sample_time_s)] 过程方向: 若 |direction_corr| ≥ 0.08: direction_corr > 0 → positive_gain direction_corr < 0 → negative_gain 否则 uncertain
方向置信度: direction_strength = |direction_corr| excitation_factor = min(1, mv_excitation_ratio / 0.12) consistency_bonus = 0.15 当 level_corr 与 diff_corr 同号且二者 |corr| ≥ 0.12 conflict_penalty = 0.25 当 level_corr 与 diff_corr 反号且二者 |corr| ≥ 0.12 direction_confidence = clip(((direction_strength - 0.08) / 0.42) × excitation_factor + consistency_bonus - conflict_penalty, 0, 1)
专家核对点：当前方向判断是统计相关法，适合历史数据初筛，不等同于严格开环阶跃辨识。若存在多扰动、串级、前馈、SP 同步变化，方向置信度可能偏低或冲突。

4.6 SP 跟踪指标

误差: e(k) = PV(k) - SP(k) 平均误差: mean(e) 绝对误差均值: mean(|e|) RMS误差: sqrt(mean(e²)) IAE: Σ |e(k)| ITAE: Σ k × |e(k)| 误差落入比例: mean(|e| ≤ 1%/2%/5% × PV_span)

若没有 SP/SV，则返回 sp_available=false，不强行评估跟踪误差。

4.7 事件与激励特征

通用大变化阈值: threshold(diff) = max(floor, percentile(|diff|, 95), 6 × MAD(|diff|)) PV 大变化计数: count(|diff(PV)| ≥ pv_threshold) MV 相邻跳变计数: count(|diff(MV)| ≥ mv_threshold) MV 斜坡事件: 在 300s 窗口内计算 |MV(k) - MV(k-ramp_n)| 阈值 = max(0.5, percentile(MV_ramp, 95))

4.8 约束与饱和 constraint_raw

指标公式/规则用途
MV 低限/高限饱和 MV ≤ 1.0 或 MV ≥ 99.0 当前默认假设 MV 是 0~100% 开度/输出。
MV 饱和比例 mean(MV≤1 or MV≥99) 用于监控、准入、窗口质量惩罚。
最长 MV 饱和时间 连续饱和段点数 × 采样周期 长时间饱和说明阀位或控制余量不足。
PV/SP 近观测边界比例 低于 P1 或高于 P99 的比例 由于无真实量程时，以观测边界近似。

4.9 频谱与振荡

实现函数：_frequency_raw、_oscillation_raw。

去趋势: baseline = rolling_median(x, window≈1800s) x_det = x - baseline FFT 功率: power(f) = |rFFT(x_det)|² dominant_frequency = argmax(power), 忽略 DC 分量 dominant_period_s = 1 / dominant_frequency power_ratio = max(power) / Σ power 零交叉次数: zero_crossings = count(sign(x_det[k]) != sign(x_det[k-1]))
振荡判定: 若 PV 主频周期存在 且 PV 主频功率占比 ≥ 0.22 且 PV 零交叉 ≥ 4/h: detected = true confidence = min(1, 0.45 + power_ratio + min(0.25, zero_crossing_per_hour/120)) severity = alarm 当 power_ratio ≥ 0.45 且 zero_crossing ≥ 10/h，否则 warning 弱振荡: 若 power_ratio ≥ 0.18 且 zero_crossing ≥ 8/h: detected = true confidence = min(0.75, 0.35 + power_ratio + min(0.2, zero_crossing_per_hour/180))

相位提示 phase_hint：若 PV/MV 主周期相差不超过 20% 且 MV 主频功率≥0.12，则为 pv_mv_same_period；若 MV 主频弱，则为 pv_only_periodic；否则 unknown。

4.10 噪声与 SNR

噪声基线: baseline = rolling_median(x, window≈300s) residual = x - baseline 噪声标准差: noise_residual_std = std(residual) 噪声比例: noise_ratio = noise_residual_std / span(x) SNR: snr_db = 20 × log10(std(x) / std(residual))

SNR 单位 dB 是对数比值单位，不是物理量单位。数值越大，说明总信号波动相对高频残差越强；数值越小，说明噪声占比越高。

4.11 Harris 指数、Cpk、振荡指数

实现函数：_performance_raw。

当前 Harris 近似实现: 若 SP 有效: e = PV - SP 若 SP 常值: e = PV - median(SP) 若无 SP: e = detrended(PV) actual_var = var(e) benchmark_var = pv_noise_residual_std² harris_index = clip(benchmark_var / actual_var, 0, 1) harris_degradation_index = 1 - harris_index
专家核对点：当前 Harris 是“高频噪声方差近似最小可达方差”的简化指标，不是经典 Harris 最小方差控制基准。经典做法通常需要估计死区和预测模型，以预测误差方差或最小方差控制性能作为基准。若要严格实现，可按“互相关估 deadtime → ARX/XGBoost/FOPDT 预测 PV(k+d) → Var(prediction error) / Var(PV 或误差)”重构。
Cpk: 需要数据中存在 PV_LSL/PV_LL/PV_LOW/PV_LOWER_LIMIT/LSL 和 PV_USL/PV_HH/PV_HIGH/PV_UPPER_LIMIT/USL 等列。 cpk = min((USL - mean(PV)) / (3 × std(PV)), (mean(PV) - LSL) / (3 × std(PV)))

若上下规格限不存在，则 cpk=null，前端应显示“缺少规格限”。

4.12 阀门/执行机构特征

实现函数：_actuator_profile、_lagged_deadband_profile、_hysteresis_profile。

指标公式/规则说明
MV 分辨率估计 percentile(|dMV| where |dMV|>0, 5) 近似执行器最小可见动作。
小 MV 动作无 PV 响应比例 小动作：|dMV| ≤ max(2×resolution, 0.005×MV_span)；无响应：|dPV| ≤ 4×PV_noise_std 死区迹象，但只基于相邻点。
滞后死区比例 按回路类型设响应滞后：flow 10s、pressure 60s、temperature 300s、level 600s；在滞后窗口内若 MV range 足够而 PV range ≤ noise_thr，则记为死区证据。 比相邻点更接近过程实际响应时间。
死区宽度估计 死区证据窗口 MV range 的 P70。 作为经验宽度，不等同阀门实际机械死区。
回差 hysteresis 分别统计 MV 上行/下行时的增益 gain=ΔPV/ΔMV，ratio = |up_gain - down_gain| / |median(all_gain)|。 ratio ≥ 0.5 标记回差迹象。
粘滞/卡涩 若 MV 不活跃连续时间 ≥ max(3600s, 120×sample_time) 且有过动作，则提示 stiction。 当前是历史统计提示，不是阀门专用回环试验。
速率限制迹象 p99_abs_step / p95_abs_step < 1.5 且动作次数>20。 若大多数动作幅值接近，可能存在速率/步长限制。

5. 窗口候选算法族与评分公式

当前推荐入口是 providers/window/policy_composite.py，由本体/LLM 生成的策略控制算法族优先级、窗口长度、扫描参数和质量门槛。

5.1 算法族

算法族Provider候选逻辑典型用途
SP 阶跃 sp_step SV/SP 存在且变化明显时，检测 SP 阶跃。 设定值阶跃试验或操作员给定变化。
MV 阶跃 mv_step 相邻 MV 跳变超过阈值，按 top-k 和间隔去重。 手动输出阶跃、阀位快速改变。

5.2 窗口质量 score_window

实现：algorithms/data_analysis.py::score_window。虽然目前代码有历史注释乱码，但公式逻辑清晰。

鲁棒噪声: noise(x) = median(|diff(x) - median(diff(x))|) MV 有效激励: mv_eff = MV_span ≥ max(12 × mv_noise, 1e-6) 且 std(MV) ≥ max(4 × mv_noise, 1e-6) PV 有效响应: pv_eff = PV_span ≥ max(10 × pv_noise, 1e-6) 且 std(PV) ≥ max(3 × pv_noise, 1e-6)
滞后相关: 对 lag = 0...min(N/4, 50) 搜索: corr(lag) = mean((MV-mean) × lagged(PV-mean)) / (std(MV) × std(PV)) 当前只保留最大正相关 corr。
饱和比例: low = min(MV) + 0.01 × MV_span high = max(MV) - 0.01 × MV_span sat_ratio = max(mean(MV≤low), mean(MV≥high)) PV 漂移比例: slope = linear_fit_slope(PV vs index) drift_ratio = |slope| × (N-1) / PV_span
子分数: mv_excitation_score = 1 若 mv_eff，否则 min(0.99, max(MV_span/(12noise), std/(4noise)) × 0.5) pv_response_score = 1 若 pv_eff，否则 min(0.99, max(PV_span/(10noise), std/(3noise)) × 0.5) correlation_score = clip(corr / 0.4, 0, 1) saturation_score = 1 - min(sat_ratio/0.6, 1) × 0.7 drift_score = 1 - min(drift_ratio/1.0, 1) × 0.25 window_score = (0.4 × mv_excitation_score + 0.4 × pv_response_score + 0.2 × correlation_score) × saturation_score × drift_score

窗口可用条件：mv_eff && pv_eff && corr ≥ 0.05 && sat_ratio ≤ 0.6。策略可进一步加入 min_mv_excitation、min_pv_response、max_drift_ratio、max_mv_saturation_ratio 等门槛。

5.4 策略分数叠加

policy_weight = 0.25 + 0.25 × policy_confidence consistency 初始 = 1.0 硬拦截: - 算法族被禁用 - 点数 < min_window_points - MV_span < min_mv_excitation - PV_span < min_pv_response 软扣分: - 算法族降级或非优先 - 点数 > max_window_points - 饱和比例 > max_mv_saturation_ratio - 漂移比例 > max_drift_ratio policy_score = original_score × (1 - policy_weight) + consistency × policy_weight usable_after_policy = original_usable && !hard_block && consistency ≥ 0.45

6. 系统辨识模型、仿真公式与评分

核心实现：backend/core/algorithms/system_id.py；provider 包装：backend/core/providers/identification/transfer_function_fit.py。

6.1 支持模型

模型传递函数/离散仿真参数参数个数 k
FO G(s)=K/(Ts+1) K, T 2
FOPDT G(s)=K·e^(-Ls)/(Ts+1) K, T, L 3
SOPDT G(s)=K·e^(-Ls)/((T1s+1)(T2s+1)) K, T1, T2, L 4
IPDT G(s)=K·e^(-Ls)/s K, L 2
SOPDT_UNDER G(s)=K·e^(-Ls)/(T²s²+2ζTs+1) K, T, ζ, L 4
IFOPDT G(s)=K·e^(-Ls)/(s(Ts+1)) K, T, L 3

6.2 数据归一化

以窗口首点为基准: PV_dev = PV - PV[0] MV_dev = MV - MV[0] 尺度: PV_scale = max(|PV_dev|) MV_scale = max(|MV_dev|) 归一化: pv_norm = PV_dev / PV_scale mv_norm = MV_dev / MV_scale 物理增益: K_physical = K_fitted × PV_scale / MV_scale

这一点很重要：拟合优化是在归一化空间进行，最终 K 会换算回工程量单位。

6.3 死区/时滞初估

最大搜索滞后: max_lag = max(3, min(round(60/dt), N/4)) 互相关: score(lag) = mean(MV_centered[0:N-lag] × PV_centered[lag:N]) / (std(MV) × std(PV)) 若 best_score > 0.1: L_hint = best_lag × dt 否则: L_hint = 0

6.4 离散仿真公式

FO / FOPDT: alpha = dt / (T + dt) d = round(L / dt) u = MV[i-d] y[i] = (1 - alpha) × y[i-1] + K × alpha × u
SOPDT 串联两个一阶: a1 = dt / (T1 + dt) a2 = dt / (T2 + dt) y1[i] = (1-a1)y1[i-1] + a1 u y2[i] = (1-a2)y2[i-1] + a2 y1[i] y[i] = K × y2[i]
IPDT: y[i] = y[i-1] + K × dt × u_delayed
SOPDT_UNDER 欠阻尼: a = T²/dt² + 2ζT/dt + 1 b1 = 2T²/dt² + 2ζT/dt b2 = T²/dt² y[i] = (K×u + b1×y[i-1] - b2×y[i-2]) / a 当前 ζ 优化边界为 [0.05, 0.99]
IFOPDT: alpha = dt / (T + dt) y1[i] = (1-alpha)y1[i-1] + alpha u y[i] = y[i-1] + K × dt × y1[i]

6.6 拟合指标

SSE = Σ (y_true - y_pred)² SST = Σ (y_true - mean(y_true))² R² = 1 - SSE / SST NRMSE = sqrt(mean((y_true - y_pred)²)) 注意：当前是在归一化 PV 上计算，名称为 normalized_rmse。
AIC: AIC = n × ln(SSE/n) + 2k 代码等效使用 NRMSE² 作为 SSE/n 的近似。 fit_score: fit_score = 10 × (0.6 × max(R²,0) + 0.4 × max(1-NRMSE,0)) - 0.005 × AIC

若 FO/FOPDT/SOPDT/SOPDT_UNDER 的 T 低于回路最小合理 T，会额外扣 100 分。IPDT/IFOPDT 当前未按相同逻辑做 T 下界惩罚。

6.7 模型置信度

short_factor = clip((200-N)/150, 0, 1) r2_weight = max(0.3, 0.65 - 0.2 × short_factor) rmse_weight = 1 - r2_weight rmse_ref = max(0.35, 0.45 if N<200) rmse_score = max(0, 1 - (NRMSE/rmse_ref)^1.2) confidence = r2_weight × clip(R²,0,1) + rmse_weight × rmse_score

随后根据漂移比例、饱和比例、低 R² 做降权或封顶：R²<0.4 时置信度≤0.45，R²<0.25 时≤0.30。

8. 闭环仿真与性能评估

核心实现：algorithms/pid_evaluation.py；provider 聚合：providers/evaluation/closed_loop_sim.py。

8.1 闭环仿真

仿真设定: SP: 50 → 60，反向校验 60 → 50 step_t = 10 MV 初值: mv0 = clip(50 - ΔSP/(|K|+eps) × 0.3, 5, 95) PID: error = SP - PV integral += error × dt derivative = (error - previous_error) / dt MV = clip(mv0 + Kp×error + Ki×integral + Kd×derivative, 0, 100)

过程模型仿真使用辨识模型的 FO/SOPDT/IPDT 等离散公式。仿真步长为 sim_dt=max(0.05, min(csv_dt, T1/10))，避免快过程被原始采样周期混叠。

8.2 响应指标

稳态误差: tail = 最后 10% 响应 SSE_percent = |mean(tail) - SP_final| / |ΔSP| × 100% 超调: 正阶跃: max(0, (max(PV)-SP_final)/ΔSP ×100%) 反阶跃: max(0, (SP_final-min(PV))/|ΔSP| ×100%) 调节时间 settling_time: 误差进入 ±2%×|ΔSP| 并保持到末尾所需时间。 若未进入保持区间，返回 -1。 振荡次数: 误差信号围绕 SP 变号次数 / 2。 衰减比: 同向超调峰值 peak2 / peak1。

8.3 性能评分 performance_score

子项权重打分规则摘要
超调25%≤2% 得 10，≤5% 得 9，≤10% 得 8，≤15% 得 7，≤25% 得 6，≤40% 得 4，继续下降。
调节时间20%≤15s 得 10，≤30s 得 9，≤60s 得 7.5，≤120s 得 6，≤300s 得 4，≤600s 得 2，未稳定/更慢得低分。
稳态误差25%≤0.5% 得 10，≤1% 得 9，≤2% 得 7.5，≤5% 得 5.5，继续下降。
振荡次数15%0 次得 7，1~2 次得 10，3~4 次得 7，过多扣分。
衰减比15%≤0.1 得 10，≤0.25 得 9，≤0.5 得 6，≤0.8 得 3，>1 得 0。
raw_score = 0.25×overshoot_score + 0.20×settling_score + 0.25×sse_score + 0.15×oscillation_count_score + 0.15×decay_ratio_score 若不稳定，按问题项软扣分，最多扣 4 分。

8.5 鲁棒、约束、就绪与最终评分

鲁棒评分: 对模型做三组扰动: K/T ±10%，L ±20% robustness_score = 0.6 × min(scores) + 0.4 × mean(scores) 约束评分: sat_pct = MV≤0.5 或 MV≥99.5 的比例 mv_tv = Σ |diff(MV)| constraint_score = 10 - min(5, sat_pct/8) - min(3, mv_tv/250) - 2.5 若正向仿真不稳定 readiness_score = 0.45×performance_score +0.20×reverse_score +0.20×robustness_score +0.15×constraint_score final_rating = 0.7×performance_score + 0.3×confidence×10

8.6 Reality Check 与封顶

Reality Check 使用回路类型典型时间常数再仿真一次，避免“模型把 T 拟合得不合理但闭环自评很高”。

回路典型 T 范围

pressure15 ~ 120s
temperature120 ~ 1200s
level300 ~ 1800s
typical_T: 若 identified_T < 下界 → 下界 若 identified_T > 上界 → 上界 否则: typical_T = clip(identified_T × (1.15 + 0.55×(1-confidence)), 下界, 上界) reality_diverged: (performance_score - reality_score) > 3.0 或 reality_sim.is_stable == false
评分封顶: confidence < 0.5: performance/final/readiness ≤ 5.0 reality_diverged = true: performance/final/readiness ≤ 4.0 tuning_unreliable = true: performance/final/readiness ≤ 3.0

9. LLM、本体/MCP 与策略评审

9.1 本体/MCP 查询

本体上下文获取在 pipeline/ontology_mcp_context.py，配置在 core/mcp_config.py 和 api/mcp_config_routes.py。窗口候选阶段会针对当前回路向 MCP/本体查询：

查询问题示例: 查询 PID 回路 {loop_id} 的本体知识： 变量角色、干扰变量、最小阶跃幅值、噪声容忍度、 预期时滞/时间常数、增益方向、 应优先或避开的窗口特征。

若 MCP 返回内容可用，会进入 ontology_mcp_content_raw / mcp_context.content，再被策略生成和 LLM 评审使用。

9.2 窗口策略生成

默认规则策略在 ontology_policy_builder.py；如 LLM 可用，window_policy_advisor.py 会基于 LoopFeatures 原始画像、本体内容、基础策略生成 JSON 策略。

LLM 被要求只输出 JSON，字段包括算法族优先级、最小激励、最大饱和、最大噪声、预期 T/L、窗口长度、扫描步长、候选数量、准入/避开工况、理由和本体证据。

9.3 LLM 是否直接改变候选窗口？

不是直接替代算法选窗。当前 LLM 的作用是生成策略 JSON；确定性窗口算法族读取策略字段后执行，再将候选窗口交给 LLM 做可解释评审。这样既避免 LLM “凭空造窗口”，也让本体知识进入确定性算法输入。

9.4 辨识结果 LLM 评审与精修

模型评审：pipeline/identification_advisor.py。当前只允许输出 accept 或 downgrade，不再直接 reject 中止。精修建议：pipeline/identification_refinement_advisor.py，让 LLM 决定下一轮是否换窗口、缩模型池、给 L 初值提示等。

闭环: 初轮辨识 attempts → LLM 评审模型可信度 → 若 downgrade，生成 refinement: force_window_index? force_model_types? hint_L? → 再辨识 → 最终选择进入整定

模块当前实现建议核对/可能优化
Harris 指数 用 PV 高频残差方差近似最小方差基准。 若要严格 Harris，应增加死区估计 + 预测模型 + 预测误差方差基准。
过程方向 基于 PV/MV 滞后相关和差分相关。 建议结合本体变量角色、控制器正反作用、窗口内阶跃方向做一致性校验。
窗口质量 MV激励、PV响应、滞后相关、饱和、漂移综合评分。 建议确认权重 0.4/0.4/0.2 及 corr≥0.05 的门槛是否适合现场历史数据。

时间常数下界 辨识优化器 T 下界按回路类型与采样周期取 max。 本体返回的预期 T/L 可进一步用于模型参数 bounds，而不只是窗口长度。

执行机构诊断 基于历史相关统计推断死区、回差、粘滞、速率限制。 建议标为“迹象/提示”，不要等同阀门专用测试结论。
